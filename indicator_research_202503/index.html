Research App

Indicator Research PoC and Evaluation Workbench

A side-by-side experiment for evaluating TradingView embedding versus dashboard-native indicator extraction before integrating into the Macro & Tech Intelligent Dashboard.

Recommended Direction

Dashboard-owned values first Use visual embeds for rapid evaluation, but keep metric extraction and persistence under app control.

What this PoC covers

  • Embeddable TradingView Advanced Chart exploration
  • Dashboard-native metric extraction with Alpha Vantage benchmark data
  • Indicator calculation examples for SMA, EMA, and RSI
  • Evaluation notes and decision criteria saved locally

How to use this experiment

  1. Start with the widget tab to test visual embedding.
  2. Move to the dashboard-native tab to inspect extracted values.
  3. Record feasibility notes and recommendations in the evaluation tab.

Option A

TradingView widget

Best for quick chart rendering, chart-type switching, and visual studies evaluation with minimal local code.

  • Fastest proof of concept
  • Good for layout and chart interaction testing
  • Weakest path for dashboard-owned metric extraction

Option B

Dashboard-native chart + data pipeline

Best for values that need to appear in your own dashboard fields, persistence model, and future automation workflows.

  • Supports explicit metric snapshots
  • Lets the app own calculations and refresh rules
  • Requires an external market-data source in a future build

Option C

Charting Library

Most control, but heavier licensing, datafeed, and integration requirements. Keep as a deeper investigation track.

  • Rich chart control and overlays
  • Likely highest setup and governance cost
  • Not required for first evaluation pass

Integration seam in the existing dashboard

The first implementation target should be the expanded macro indicator detail view, which already exists in the dashboard. This experiment mirrors that detail-module approach instead of proposing a separate production route.

Widget research notes

  • Best suited to visual proof of concept and layout validation.
  • Study selection is supported at render time through widget config.
  • Do not treat widget rendering as proof of dashboard-owned metric extraction.
  • If the chart does not load, check network access to TradingView script hosts.
Important: This tab intentionally does not claim ownership of structured indicator values. That capability is evaluated in the dashboard-native tab.
